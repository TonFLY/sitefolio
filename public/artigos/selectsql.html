Dica de SQL: Por que evitar SELECT *?

Publicado em 09 de maio de 2025

Se você está começando a escrever consultas SQL, é comum usar SELECT * para buscar todos os dados de uma tabela. Mas essa prática, embora pareça conveniente, pode trazer diversos problemas.

🧠 O que o SELECT * faz?

Ele traz todas as colunas de uma tabela — até mesmo aquelas que você não vai usar. Isso pode:

  • ⚠️ Deixar a consulta mais lenta
  • 📉 Prejudicar a legibilidade do código
  • 🔐 Expor dados sensíveis por engano

✅ Como fazer melhor?

Especifique apenas as colunas que você realmente precisa:

-- Evite
SELECT * FROM clientes;

-- Prefira
SELECT nome, email FROM clientes;

⚙️ Por que isso melhora a performance?

Bancos de dados usam estruturas chamadas índices para encontrar dados rapidamente. Quando você pede tudo (*), o banco pode ignorar esses índices e ler muito mais dados do que o necessário. Isso impacta:

  • 🚫 O tempo de resposta da consulta
  • 🚫 O uso de memória
  • 🚫 A reutilização de cache

🍽️ Uma analogia rápida

Usar SELECT * é como ir num buffet e encher o prato com tudo, até o que você não vai comer. Já escolher colunas específicas é como pedir à la carte: só vem o que você quer, do jeito certo, e sem desperdício.
